My First NEW Mind Map

微架构功能利用
进展：传统Spectre攻击、Stale I/D cache（利用Cache/TLB等传统侧信道+部分微架构训练效果）
微架构结构分析
问题：顶会文章倾向于发现一种新的观测方法，但是鉴于Spectre出现至今x86上没有相关研究，该方向难度较大--可以从ISA特性出发。
进展：RISC-V领域还未出现顶会论文，都是基于BOOM模拟器的文章（分析内部的结构）;x86和ARM上论文很多，而且这是共有问题，迁移性更高。
问题： 1. 结构分析比较困难（某些预测结构可能存在时间窗口较短）、无法精确做到Gadget定位（攻击一般是user->kernel,需要kernel侧存在Gadget且user侧控制跳转到指定Gadget） 2. x86上基于新的硬件特性结合旧的漏洞可以发现新的问题，但是RISC-V新特性推出过慢--如果要做只能考虑模拟器
RISC-V微架构攻击
已有分析
1. RAS（ret指令）跨特权漏洞大概率不存在（预取return时会考虑当前特权级） 2. IBP（间接跳转指令）在同特权级可以指定替换并且瞬态执行，跨特权级可以影响，但精确的index替换逻辑需要进一步分析 （2022Usenix-Branch History Injection: On the Effectiveness of Hardware Mitigations）
1. 2023SP-A Security RISC: Microarchitectural Attacks on Hardware RISC-V CPUs 2. 2025CCS-GhostCache: Timer- and Counter-Free Cache Attacks ExploitingWeak Coherence on RISC-V and ARM Chips 3. 未发表RISCY Cache 这些论文在开发板上可以直接复现，真实攻击场景不跨特权级别，主要是AES密钥读取/Spectre泄密
1. 2023MICRO-Phantom: Exploiting Decoder-detectable Mispredictions 2. 2022Usenix-RETBLEED: Arbitrary Speculative Code Execution with Return Instructions
