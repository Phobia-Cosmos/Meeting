# 组会汇报：HTML / Markdown / PPTX 的关系与推荐工作流

## 1. 先说清楚：`PPTX` 不是 `HTML`

- `PPTX` 本质上是一个包含多份 XML 的压缩包，是 PowerPoint 的原生格式。
- `HTML` 本质上是网页，可以在浏览器里显示“像 PPT 一样”的幻灯片。
- 所以大家常说的“HTML 类型 PPT”，严格来说通常是指：**用 HTML 作为幻灯片的展示格式或中间格式**，而不是说 `.pptx` 文件本身就是 HTML。

## 2. 常见的三种做法

### 方案 A：直接做 HTML 幻灯片

适合：先快速预览、修改内容结构、确认讲述逻辑。

工作方式：

1. 先把你每周目录里的图片、结果、截图、笔记整理出来。
2. 我帮你分析哪些内容适合汇报。
3. 先生成一份结构化内容（Markdown 或提纲）。
4. 再生成一份 HTML 幻灯片。
5. 你可以直接在浏览器里放映。

优点：

- 改内容很快。
- 样式统一容易。
- 适合先快速出一版。

缺点：

- 不是原生 PowerPoint。
- 如果一定要交 `.pptx`，还需要额外转换。

### 方案 B：Markdown / 结构化内容作为“源”，同时生成 HTML 和 PPTX

适合：你以后每周都要重复做，想把流程标准化。

工作方式：

1. 每周资料放到对应目录，比如 `03-14/`。
2. 我先读取该目录的图片、文档、日志、结果截图。
3. 我提炼出：
   - 本周目标
   - 已完成工作
   - 关键结果
   - 问题与分析
   - 下周计划
4. 先生成 `report.md`。
5. 再由 `report.md` 生成 `report.html`。
6. 如你需要，我再继续生成 `report.pptx`。

优点：

- 最适合长期使用。
- 内容与展示分离，后期维护方便。
- 同一份内容可以导出成多种格式。

缺点：

- 需要先约定模板。

### 方案 C：把 HTML 页面“截图式”塞进 PPTX

适合：只追求版式接近 HTML 页面，不要求 PowerPoint 内部元素可编辑。

优点：

- 视觉一致。

缺点：

- 可编辑性差。
- 后期在 PowerPoint 里改单页内容比较麻烦。

## 3. 我推荐你的方案

对你的场景，我建议用：

**每周资料目录 -> 内容分析 -> `report.md` -> `report.html` -> 需要时再导出 `report.pptx`**

这是最稳妥的方式，因为你每周都会重复做组会汇报。

## 4. HTML 幻灯片是怎么“工作”的

一个 HTML 幻灯片文件通常由三部分组成：

### 内容层

- 每一页就是一个 `<section>` 或 `<div>`。
- 一页里包含标题、要点、图片、结论等。

### 样式层

- 用 CSS 控制 16:9 比例。
- 控制字体、颜色、边距、图文布局。
- 控制打印时一页一页分页。

### 交互层

- 用少量 JavaScript 控制上一页、下一页。
- 也可以加页码、目录、高亮动画。

为了让我自动分析并生成更好的汇报，建议每周目录尽量包含：

- 实验截图
- 核心结果图
- 表格或数据导出结果
- 代码运行日志或结论摘要
- 你自己的一两句备注（哪怕很短）
